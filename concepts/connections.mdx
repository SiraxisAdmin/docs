## Using Connections

Creating a connection is your starting point and defines the **account endpoint and key** that can be used for managing container definitions and optional query definitions.

<Tip>
**READ/WRITE** keys are required for inserting, updating, or deleting container data.
</Tip>
<Tip>
**READONLY** keys are sufficient if you just want to read container data.
</Tip>
<Note>
**ACCOUNT ENDPOINTS** must be publicly accessible from within Azure.
</Note>
